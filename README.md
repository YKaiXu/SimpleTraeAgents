# SimpleTraeAgents

Trae 国内版自定义智能体定义文档集

> **制定原则**：简约高效 — 每个智能体聚焦单一职责，提示词精炼，调用场景明确。

## 概述

本仓库定义了 **6 个** 面向 Trae 国内版的自定义智能体，覆盖软件开发全流程：**调研 → 设计 → 全栈开发 → 测试 → 审查 → 文档**。

每个智能体都具备：
- **工具声明** — 明确告知智能体能使用哪些工具（阅读、编辑、终端）
- **项目规范感知** — 调用时先读取 `.trae/rules/` 了解项目约定
- **整体规划感知** — 读取 `.trae/specs/` 了解项目整体设计
- **结构化的输出格式** — 使用模板约束输出，保证结果一致可用
- **SOLO Agent 协作** — 可被 SOLO Agent 自动编排调用

## 智能体一览

| 智能体 | 英文标识名 | 职责 | 哲学来源 | SOLO Agent 调用时机 |
|---|---|---|---|---|
| [调研智能体](research-agent.md) | `research-agent` | 需求调研、文档研究、技术选型分析 | — | 复杂任务开始前的前置调研 |
| [UI 设计智能体](ui-designer.md) | `ui-designer` | 生成 UI 代码、图片资源、响应式布局 | Trae text-to-image | 需要生成或修改用户界面时 |
| [Python 全栈智能体](python-fullstack.md) | `python-fullstack` | Python Web 全栈开发（Django / FastAPI / Flask） | Linus / Reitz / Ronacher / Hettinger | 需要开发 Python Web 全栈项目时 |
| [代码审查智能体](code-reviewer.md) | `code-reviewer` | 代码质量检查、Bug/安全/性能审查 | Linus Torvalds P0-P7 | 代码变更完成后审查 |
| [测试工程师智能体](test-engineer.md) | `test-engineer` | 编写单元测试、集成测试，运行验证 | — | 代码编写完成后，根据 specs 编写测试 |
| [技术文档智能体](tech-writer.md) | `tech-writer` | 生成 README、API 文档、架构文档、变更日志 | — | 交付前生成文档 |

```text
用户需求
    │
    ▼
┌──────────────────┐
│  调研智能体       │  Step 1: 前置调研
│  research-agent   │  输出：调研报告（含技术选型建议）
└────────┬─────────┘
         │ 调研报告
         ▼
┌──────────────────┐
│  SOLO Agent      │  Step 2: 制定计划（Plan/Spec）
│  （主控，内置）   │  参考：.trae/specs/ + 调研报告
└────────┬─────────┘
         │ 计划确认
         ▼
┌──────────────────────────────────┐
│  Step 3a: UI 设计 (ui-designer)  │  ← 如需 UI
│  或                            │
│  Step 3b: Python 全栈开发          │  ← Python 项目
│           (python-fullstack)      │
└────────────────┬─────────────────┘
                 │
    ┌────────────┼────────────┬──────┐
    ▼            ▼            ▼      ▼
┌────────┐┌─────────┐┌─────────┐┌────────┐
│ 测试    ││ 代码     ││ 文档     ││ 可选    │
│ 工程师   ││ 审查     ││ 智能     ││ 循环   │

```

## 配置方法

在 Trae 中创建自定义智能体时：

1. 在 AI 对话输入框中输入 `@`，点击「创建智能体」
2. 填写智能体名称（如：调研智能体）
3. 粘贴本文档中各智能体的 **提示词**（Prompt 区域）
4. **开启「可被其他智能体调用」**
5. 填写 **英文标识名**（如 `research-agent`）
6. 填写 **何时调用**（使用本文档中各智能体「SOLO Agent 自动调用」的内容）
7. 配置工具集：
   - 调研智能体：需要「阅读」工具
   - UI 设计智能体：需要「阅读」+「编辑」工具
   - Python 全栈智能体：需要「阅读」+「编辑」+「终端」工具
   - 代码审查智能体：需要「阅读」工具
   - 技术文档智能体：需要「阅读」+「编辑」工具
   - 测试工程师智能体：需要「阅读」+「编辑」+「终端」工具
8. 在 SOLO Agent 配置面板中，将自定义智能体添加到可调用列表
