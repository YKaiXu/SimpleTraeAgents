# SimpleTraeAgents

Trae 国内版自定义智能体定义文档集

> **制定原则**：简约高效 — 每个智能体聚焦单一职责，提示词精炼，调用场景明确。

## 概述

本仓库定义了 4 个面向 Trae 国内版的自定义智能体，覆盖软件开发全流程：调研 → 编码 → 测试 → 审查 → 文档。

这些智能体可以：
- 在 **IDE 模式** 下通过 `@` 手动选择使用
- 在 **SOLO 模式** 下由 SOLO Agent 自动编排调用

## 智能体一览

| 智能体 | 英文标识名 | 职责 | 调用时机 |
|---|---|---|---|
| [调研智能体](research-agent.md) | `research-agent` | 需求调研、文档研究、技术选型分析 | 复杂任务开始前的前置调研 |
| [代码审查智能体](code-reviewer.md) | `code-reviewer` | 代码质量检查、Bug/安全/性能审查 | 代码变更完成后，提交前 |
| [技术文档智能体](tech-writer.md) | `tech-writer` | 生成 README、API 文档、架构文档 | 功能开发完成后，交付前 |
| [测试工程师智能体](test-engineer.md) | `test-engineer` | 编写单元测试、集成测试，保障质量 | 代码编写完成后，审查前 |

## 智能体协作流程

```
用户需求
    │
    ▼
┌──────────────┐
│ 调研智能体    │  ← 第一步：调研需求与现有代码
│ research-agent│
└──────┬───────┘
       │ 调研报告
       ▼
┌──────────────┐
│ SOLO Agent   │  ← 主控智能体（Trae 内置）
│ (主控)       │     编排子智能体协作
└──┬───┬───┬──┘
   │   │   │
   ▼   ▼   ▼
┌────┐┌────┐┌──────────┐
│代码 ││文档 ││ 测试      │
│审查 ││智能 ││ 工程师    │
│智能 ││体   ││ 智能体    │
│体   ││     ││           │
└────┘└────┘└──────────┘
```

## 配置方法

在 Trae 中创建自定义智能体时：

1. 在 AI 对话输入框中输入 `@`，点击「创建智能体」
2. 填写智能体名称
3. 粘贴本文档中各智能体的 **提示词**
4. **开启「可被其他智能体调用」**
5. 填写 **英文标识名**（如 `research-agent`）
6. 填写 **何时调用**（使用本文档中各智能体「SOLO Agent 自动调用」的内容）
7. 配置工具集（建议：调研/审查 + 阅读工具，测试/文档 + 编辑工具）
8. 在 SOLO Agent 配置面板中，将自定义智能体添加到可调用列表
